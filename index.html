Step 01

The Big Picture

Your robot uses a 2-motor tank drive setup. Two motors power the left and right wheels independently, controlled by a VEX V5 controller with two joysticks.

⚙

2 Motors

Left motor on Port 1, right motor on Port 2. Simple but effective.

🎮

Tank Drive

Each joystick controls one side of the robot. Push both forward to go straight.

⚡

V5 Brain

The brain runs Python code and talks to motors & the controller wirelessly.

🔄

Control Loop

Your code runs 50 times per second (every 20ms), constantly reading inputs.

Step 02

Robot Configuration

Before driving, you need to tell the brain what's plugged in and where. This code sets up the brain, controller, and both motors.

robot_config.py 
# Import the VEX library
from vex import *

# Create the brain — the robot's computer
brain = Brain()

# Set up the controller
controller_1 = Controller(PRIMARY)

# Left motor — Port 1, spinning forward
left_motor = Motor(Ports.PORT1, GearSetting.RATIO_18_1, False)

# Right motor — Port 2, reversed (so both sides drive correctly)
right_motor = Motor(Ports.PORT2, GearSetting.RATIO_18_1, True)

Why is the right motor reversed?

Motors on opposite sides of the robot face opposite directions. If both spin "forward," one side goes forward and the other goes backward — making the robot spin! Setting the right motor to True (reversed) fixes this so both sides drive the same way.

Step 03

Controller & Joysticks

In tank drive, each joystick controls one side of the robot. The left stick (Axis 3) controls the left motor, and the right stick (Axis 2) controls the right motor.

VEX V5 Controller

↑↓

Axis 3 — Left Stick

Push up/down to control left motor speed. Returns -100 to 100.

↑↓

Axis 2 — Right Stick

Push up/down to control right motor speed. Returns -100 to 100.

Step 04

The Drive Loop

This is the heart of the code. It runs forever, constantly reading the controller and updating the motors. Here's how the loop works:

tank_drive.py 
# Tank drive control
while True:
    # Read joystick positions (-100 to 100)
    left_speed = controller_1.axis3.position()
    right_speed = controller_1.axis2.position()

    # Spin motors at joystick speed
    left_motor.spin(FORWARD, left_speed, PERCENT)
    right_motor.spin(FORWARD, right_speed, PERCENT)

    # Small delay — runs 50 times per second
    wait(20, MSEC)

How the Loop Flows

1. Read Left Stick

Get Axis 3 position → left_speed

↓

2. Read Right Stick

Get Axis 2 position → right_speed

↓

3. Spin Motors

Set left & right motor speeds

↓

4. Wait 20ms

Pause briefly, then repeat forever

Step 05

How to Drive

Combine both sticks to move in different ways. Here's a cheat sheet:

↑

Drive Forward

Push both sticks up. Both motors spin forward at the same speed.

↓

Drive Backward

Pull both sticks down. Both motors spin in reverse.

↺

Turn Left

Push right stick up, left stick still (or down for a sharper turn).

↻

Turn Right

Push left stick up, right stick still (or down for a sharper turn).

⇵

Spin in Place

Push sticks in opposite directions. The robot pivots on the spot.

■

Stop

Release both sticks. Joysticks return to 0, motors stop.

Complete Program

Here's the entire program from start to finish — everything your robot needs to drive.

VEXcode Project.v5python 
# VEXcode Generated Robot Configuration
from vex import *
import urandom
import math

# Create the brain
brain = Brain()

# Configure controller and motors
controller_1 = Controller(PRIMARY)
left_motor  = Motor(Ports.PORT1, GearSetting.RATIO_18_1, False)
right_motor = Motor(Ports.PORT2, GearSetting.RATIO_18_1, True)

# Wait for sensors to initialize
wait(30, MSEC)

# Tank drive control loop
while True:
    left_speed  = controller_1.axis3.position()
    right_speed = controller_1.axis2.position()

    left_motor.spin(FORWARD, left_speed, PERCENT)
    right_motor.spin(FORWARD, right_speed, PERCENT)

    wait(20, MSEC)
